 Context

 현재 Kurly Jira Tauri v2 앱에는 CI/CD 파이프라인과 자동 업데이트 기능이 없음. Git push 시 자동으로 macOS DMG을 빌드하고, 기존 사용자의 앱이 자동으로

 ---
 1. GitHub Actions 워크플로우 (.github/workflows/publish.yml)

 - 트리거: v* 태그 push (예: v0.2.0)
 - 빌드 매트릭스: macOS aarch64-apple-darwin (Apple Silicon) + x86_64-apple-darwin (Intel)
 - 사용 Action: tauri-apps/tauri-action@v0 (공식 Tauri Action)
 - 결과물: DMG, .app.tar.gz (업데이트 번들), .sig (서명), latest.json (매니페스트)
 - GitHub Release 자동 생성 및 모든 아티팩트 업로드

 2. Tauri Updater 설정

 - tauri.conf.json에 updater 플러그인 설정 추가
 - endpoint: https://github.com/kyungtaekang-kurly/jira-dashboard-app/releases/latest/download/latest.json
 - 서명 검증용 public key 포함
 - lib.rs에서 tauri_plugin_updater::Builder::new().build() 패턴으로 등록

 3. 프론트엔드 업데이트 체크 (src/update-checker.js)

 - 앱 시작 시 onMounted에서 silent 모드로 업데이트 확인
 - 새 버전 발견 시 한국어 다이얼로그 표시 ("새 버전 X.X.X이(가) 있습니다")
 - 사용자 승인 후 다운로드 → 설치 → 자동 재시작

 4. 릴리스 프로세스

 1. 3곳의 버전 동시 변경: package.json, Cargo.toml, tauri.conf.json
 2. git commit & push
 3. git tag v0.2.0 && git push origin v0.2.0
 4. GitHub Actions 자동 실행 → Release 생성
 5. 기존 사용자 앱 실행 시 자동 업데이트 감지

 코드 변경 전 사전 작업 (로컬)

 npx tauri signer generate -w ~/.tauri/jira-dashboard.key
 - 생성된 .key.pub 내용을 tauri.conf.json의 pubkey에 입력
 - 생성된 .key 내용을 GitHub Secret에 등록

 GitHub 사이트에서 해야 할 설정

 Repository Settings > Secrets and variables > Actions 에서:
 ┌────────────────────────────────────┬────────────────────────────────────────────┐
 │            Secret 이름             │                     값                     │
 ├────────────────────────────────────┼────────────────────────────────────────────┤
 │ TAURI_SIGNING_PRIVATE_KEY          │ ~/.tauri/jira-dashboard.key 파일 내용 전체 │
 ├────────────────────────────────────┼────────────────────────────────────────────┤
 │ TAURI_SIGNING_PRIVATE_KEY_PASSWORD │ 키 생성 시 설정한 비밀번호                 │
 └────────────────────────────────────┴────────────────────────────────────────────┘
 GITHUB_TOKEN은 GitHub Actions가 자동 제공하므로 별도 설정 불필요

 Repository Settings > Actions > General:
 - Workflow permissions: "Read and write permissions" 활성화 (Release 생성을 위해)

 3. 버전 일관성 - 3곳 (package.json, Cargo.toml, tauri.conf.json)의 버전이 항상 동일해야 함
